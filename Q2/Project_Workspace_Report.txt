4. UMASK VALUE USED
----------------------------------------------------
Command: umask         (checked current value)
Output:  0022          (system default)

Command: umask 027     (set secure umask)
Output:  0027          (new value confirmed)

Explanation of umask 027:
  - 0 (special bits): no setuid/setgid/sticky bit restrictions
  - 2 (owner):        owner gets full permissions (no bits masked)
  - 7 (group):        group loses write+execute (gets read only for files)
  - 7 (others):       others lose ALL permissions (no access)

Effect: New files will be created with 640 (-rw-r-----) permissions
        New directories will be created with 750 (drwxr-x---) permissions

Observation:
The umask 027 ensures that any newly created files automatically have secure
permissions — the group can read but not write, and others have no access at
all. This is much more secure than the default 022 which allows world-readable
files. Setting umask in /etc/profile makes it persistent across sessions.

----------------------------------------------------
5. HOW PERMISSIONS PROTECT PROJECT DATA
----------------------------------------------------
Linux Permission Model (rwxrwxrwx = user|group|others):

  r (read)    = view file contents / list directory
  w (write)   = modify file / create files in directory
  x (execute) = run as program / enter directory (cd)

Security layers applied in this workspace:

  a) Principle of Least Privilege:
     Each file has only the minimum permissions required. Configuration
     files (600) can only be read/written by the owner — even group
     members cannot access sensitive settings.

  b) Group-based Access Control:
     The 750/640 permissions allow team members in the same group to
     read project files without being able to modify them, preventing
     accidental or malicious changes.

  c) No World Access:
     Removing 'others' permissions (the last triplet) ensures that users
     not part of the project group cannot access any project data, even
     if they have a shell account on the same system.

  d) Directory Execute Bit:
     The execute (x) bit on directories controls whether users can 'cd'
     into them. Without it, even if a user knows a filename, they cannot
     navigate into the directory to access it.

  e) Umask Prevention:
     The umask 027 ensures future files created in this workspace
     automatically inherit secure permissions without manual intervention.
